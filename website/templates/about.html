About the Project

The Complexity Paradox started with a simple question: as computers got exponentially more powerful, did game development get easier or harder?

The answer, backed by four decades of data, is surprising. Despite better tools, larger teams, and bigger budgets, the complexity of building a great game has only increased.

Methodology

We collected data on 100+ games spanning from the 1980s to the 2020s. For each game, we tracked:

  • Development team size at peak production
  • Total development time from concept to release
  • Final file size and storage requirements
  • Metacritic score as a proxy for quality
  • Budget estimates where publicly available

Data sources include public developer interviews, GDC postmortems, industry reports, and official documentation. When exact numbers were unavailable, we used the most reliable estimates from multiple sources.

Inspiration

This project was inspired by the contrast between early game development — where a handful of people could create genre=defining experiences in just months — and modern AAA production, where thousands work for years on a single title.

We wanted to move beyond anecdotes and see what the data actually says about this shift.
